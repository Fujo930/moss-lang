# Project memory

## Notes

- # Moss Next AI 设计和构建的实验性编程语言（当前 v0.57.1），由三个 AI（DeepSeek Codex、DeepSeek Kun、Reasonix）与人类 Fujo930 合作开发。 ## 设计目标 "长期存活的产品软件，人类和 AI 在同一个代码库上工作。" 三个核心承诺：(1) 领域状态直接命名，(2) 易出错流程显式处理失败，(3) 副作用在函数边界可见。 ## 技术架构 Python 实现（~6200 行），全栈：递归下降解析器 → 静态 checker（类型推断 + effect 追踪 + 规则纯度 + 类型存在性校验）→ 32 操作码字节码编译器 → 栈式 VM。附加：C99 零依赖 VM（500 行）、自托管 Moss 前端（~2500 行 Moss 代码改写词法/解析/检查/编译）、LSP、浏览器 IDE（Studio）、Playground、VS Code 扩展。 ## Trust Artifact — 技术创新核心 五 gate 正交验证。`moss trust file.moss` 产出 JSON bundle。五个 gate 各自信任不同神谕： 1. **Check** — Python 宿主 checker 静态分析（类型/effect/穷尽性/重复/纯度/类型存在性） 2. **Trace** — VM 运行时规则求值记录（声明规则数 vs 实际捕获事件数独立验证） 3. **Golden** — 确定性输出快照严格字符串比对 4. **Lock** — 导入图哈希匹配 moss.lock 5. **Selfhost** — Moss↔Python 解析器五维等价比较（声明数/名称/体形/元数据/表达式AST） 不是 formal proof，是 **structured evidence**。消费者命令 `moss trust-verify` 重新执行全部五 gate 并比对哈希（纯重推导模型，非签名验证模型）。 ## 三轮对抗审计 78 个攻击向量，18 个漏洞，全部修复。审计由三个不同 AI 独立执行（Reasonix 主导）。V10/V11 修复不彻底（分别引入 F3/F2），F1（trust-verify 仅比对哈希不重跑 gate）为 0.56.2 新增功能的最致命漏洞。0.57.1 全部关闭。三个保留的 WARNING（未用 effect、match 模式误用、导入循环）为设计选择，`--strict` 模式可升级为 error。 ## 威胁模型 - TCB = Python 运行时 + mosslang 包 + 自宿主模块 + golden/lock 文件 - 自托管悖论：同一 AI 写代码和写 parser_core.moss 时 oracle 独立性丧失（Thompson "Trusting Trust" 的 AI 时代映射） - F1 密码学边界：bundle 无签名，消费者防线为 `--source` 显式指定源文件 + `file_redirected` 检测 - 八个已知 CVE 模式对标：均不直接 vulnerable，但递归解析器栈耗尽（对标 OpenSSL CVE-2019-1559）和导入路径穿越（对标 npm CVE-2021-24105）需加固 ## 市面独特性 1. **Multi-gate 验证**：无同类。SLSA、cargo-vet、Sigstore、Reproducible Builds 各做一个维度，无一统一 2. **AI 设计的语言**：独一档 3. **Token 效率作为一级设计约束**：无先例。backtick 插值、pipe、arrow body、lambda 每个标注 "节省 X token" 4. **自托管作为永久验证器**：无语言实现。CSmith/EMI 是研究工具，不嵌入生产命令 ## 已产出文档 - `docs/paper-trust-artifact.md`：学术论文（7 章 20 篇参考文献） - `docs/threat-model.md`：威胁模型（含自托管悖论、F1 边界、strict 模式） - `audit-report.md`、`nuke-report.md`：三轮审计完整报告 ## 发展方向 - 0.100：Trust header comments（`// @trust 5/5` 嵌入源码，15 token 替代 2000 token 验证对话） - 长期：技术遗产被吸收而非独立语言存在。当 "AI 写的代码我能信吗" 成为默认问题，Moss 的 Trust Artifact 是第一个被引用的系统。 - AI 大众化准备：Trust API for AI、fix_hint 生成、确定性语法、AI-first 设计流程
